HTML/HTML5 本地存储

1. 介绍

HTML5中与本地存储相关的有两个重要内容:

  • Web Storage
    • sessionStorage
      • setItem/getItem/removeItem/clear
    • localStorage
      • setItem/getItem/removeItem/clear
  • 本地数据库 SQLLite
    • openDatabase
    • transaction
    • executeSql

Web Storage存储机制是对HTML4中cookies存储机制的一个改善。

cookies存储永久数据存在一下问题：

  • 大小：cookies的大小被限制在4KB。
  • 带宽：cookies是随HTTP事务一起被发送的，因此会浪费一部分发送cookies时使用的带宽。
  • 复杂性：要正确地操作cookies是很困难的。

Web Storage的两种存储方式介绍：

  • sessionStorage：将数据保存在session对象中。所谓的session,是只用户在浏览某个网站时，从进入网站到浏览器关闭所经历的这段时间，
    也就是用户浏览这个网站所花费的时间。session对象可以用来保存在这段时间内所要保存的任何数据。
  • localStorage：将数据保存在客户端本地的硬件设备中，即使浏览器被关闭了，该数据仍然存在，下次打开浏览器访问网站时仍然可以继续使用。

在HTTML5中内置了一个可以通过SQL语言来访问的本地文件型SQL数据库，被称为”SQLLite”。

使用SQLLite数据库的必要的步骤：

  1. 创建访问数据库的对象。
  2. 使用事务处理。

3.1. 本地数据库openDatabase方法

HTML5本地数据库，必须要使用openDatabase方法来创建一个访问数据库的对象。
该方法的介绍如下：

该方法使用四个参数：

  • 第一个参数为数据库名
  • 第二个参数为版本号
  • 第三个参数为数据库的描述
  • 第四个参数为数据库的大小

该方法返回创建后的数据库访问对象。如果该数据库不存在，则创建该数据库。

实际访问数据库的时候，还需要调用 transaction 方法，用来执行事务处理。

使用事务处理，可以防止在对数据库进行访问及执行有关操作的时候受到外界的打扰。

3.2.transaction方法

transaction方法使用一个回调函数作为参数。在这个函数中，执行访问数据库的语句。

3.3. 用executeSql来执行访问数据库的语句

  • 第一个参数为要执行的SQL语句
  • 第二个参数为SQL语句中所有使用到的参数的数组。在executeSql方法中，将SQL语句中所要使用到
    的参数先用”?”代替。然后依次将这些参数组成数组放在第二个参数中。tx.executeSql('UPDATE MsgData set message=? WHERE name=?',["备注1","li"]);
  • 第三个参数为执行sql语句成功时调用的回调函数。该回调函数的传递方法如下所示。function dataHandler(transaction,results){//执行SQL语句成功时的处理}
  • 第四个参数为执行SQL语句出错时调用的回调函数。该回调函数的传递方法如下：function errorHandler(transaction,errmsg){//执行sql语句出错时的处理}
